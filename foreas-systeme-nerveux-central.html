2. Communication inter-branches — les tentacules se parlent

Chaque signal d'une branche déclenche une action dans une autre

CM → Prospection
Un chauffeur VTC like/commente un post → son profil est identifié → Apollo l'enrichit (email, téléphone) → ajouté au pipeline Prospection → séquence de contact déclenchée.
Délai : <5min entre le like et l'ajout au CRM
Prospection → Rétention
Un chauffeur est converti → son profil bascule automatiquement de "lead" à "actif" → Rétention prend le relais : onboarding, monitoring, churn scoring. Le transcript de l'appel de vente est transféré pour connaître ses attentes.
Handoff automatique · contexte complet transféré
Rétention → CM
Les feedbacks des chauffeurs actifs (recueillis par Rétention) alimentent le CM : si 60% des chauffeurs adorent la feature X → CM crée du contenu autour de X. Si une objection revient → CM publie un post qui y répond préventivement.
Boucle feedback → contenu en <24h
Admin → Rétention
Détection d'un pic d'échecs de paiement → Rétention envoie un SMS ciblé "Un souci avec votre abonnement ?". Détection d'un bug impactant → Rétention appelle les chauffeurs touchés pour s'excuser et offrir une compensation.
Réaction automatique aux incidents techniques
Data → Toutes les branches
La branche Data détecte qu'un segment de chauffeurs (ex: "solo, IDF, >2 ans XP") a un taux de conversion 3x supérieur. Elle envoie cette info à Prospection (cibler ce segment), CM (créer du contenu pour ce profil), et Admin (monitorer leur rétention en priorité).
Intelligence distribuée en temps réel
Compliance → Bloque toute branche
Si CM génère un post avec une promesse de revenu non sourcée → bloqué. Si Prospection appelle un numéro sur liste rouge → bloqué. Si Data prépare un dataset avec des données non anonymisées → bloqué. Override possible uniquement par toi via Telegram.
Veto automatique · protection juridique permanente

4. Commandes Telegram — ton tableau de bord vocal/texte

Tu parles ou tu tapes, Ajnaya exécute

/status
Résumé instantané de toutes les branches. Vert = nominal, Orange = attention requise, Rouge = incident.
/cm pause
Met en pause les publications CM. Utile en cas de crise ou d'événement sensible.
/prospect volume 50
Passe le volume d'appels quotidien à 50. Ajnaya ajuste les créneaux et le budget Twilio.
/prospect cible lyon uber
Ajoute les chauffeurs Uber de Lyon au pipeline. Apollo identifie, Ajnaya appelle.
/retention rapport
Rapport détaillé rétention : chauffeurs à risque, NPS, saves, feedback récents.
/admin mrr
MRR actuel, évolution 7j, projections M+1, top 10 chauffeurs par LTV.
/data insight
Dernier insight cross-branch détecté par la branche Data. Pattern, opportunité, ou alerte.
/boost [montant]
Boost le meilleur post du jour en ads. Budget spécifié, audience auto-ciblée.
/expansion [pays]
Lance le setup d'un nouveau pays : workflows dupliqués, langue configurée, Apollo sourcé, ads préparées.
/call [nom/entreprise]
Ajnaya passe un appel immédiat au contact spécifié. Transcript en direct dans le chat.
/override [branche] [action]
Force une action qu'une branche a bloquée (compliance override). Log enregistré.
🎤 Message vocal
"Ajnaya, combien on a fait aujourd'hui ?" — Whisper transcrit, Claude interprète, réponse instantanée.

5. Plan d'action — dans quel ordre on monte tout ça

5 phases séquentielles · chaque phase finance et débloque la suivante

PHASE 1 — FONDATIONS Semaine 1-2
Mettre en place le bot Telegram DG + la branche Prospection
  • Créer le bot Telegram "Ajnaya DG" avec le framework de routing des commandes (n8n)
  • Connecter Apollo.io → n8n : premier workflow de recherche de leads chauffeurs VTC France
  • Configurer ElevenLabs Conversational AI : voix Ajnaya, prompt system commercial, RAG avec la doc FOREAS
  • Acheter le numéro Twilio (+33) et connecter à ElevenLabs
  • Créer le workflow n8n : Apollo → enrichissement Claude → appel ElevenLabs/Twilio → log Supabase → rapport Telegram
  • Premiers 10 appels tests manuellement validés
Résultat : Ajnaya passe ses premiers vrais appels. Tu reçois les rapports dans Telegram.
PHASE 2 — AMPLIFICATION Semaine 3-4
Ajouter la branche CM + connecter à Prospection
  • Configurer les comptes sociaux FOREAS (Instagram, TikTok, LinkedIn, X, YouTube, Facebook) dans n8n
  • Créer les templates Creatomate + Canva (3-4 formats par réseau)
  • Workflow CM : Claude génère → Creatomate/Canva rend → n8n publie → métriques collectées
  • Pont CM → Prospection : likes/commentaires identifiés → Apollo enrichit → pipeline prospect
  • Configurer ActiveCampaign : séquences email par segment, lead scoring, tags automatiques
  • Rapport CM intégré au rapport DG Telegram
Résultat : Publications automatiques + prospects sociaux capturés + double canal de génération de leads.
PHASE 3 — RÉTENTION Semaine 5-6
Branche Rétention + Churn Scoring + Admin
  • Créer les tables Supabase de scoring churn (sessions, Ajnaya usage, notifs, feedback, paiements)
  • Workflow churn : Supabase triggers → calcul score → seuils → action (SMS, email, appel Ajnaya)
  • Connecter Stripe → Supabase → n8n : monitoring paiements, MRR temps réel, alertes échecs
  • Workflow onboarding : inscription → SMS bienvenue → push J+1 → vérification profil J+2 → appel J+3 si incomplet
  • Dashboard admin intégré au rapport DG
Résultat : Aucun chauffeur ne churn silencieusement. Chaque décrochage est détecté et traité.
PHASE 4 — INTELLIGENCE Semaine 7-8
Branche Data + Cerveau Central + Compliance
  • Pipeline data cross-branch : agréger les métriques de CM, Prospection, Rétention, Admin dans une vue unifiée
  • Claude API analyse hebdo : détecter les patterns cross-branch, générer des insights actionnables
  • Auto-optimisation : les insights mettent à jour automatiquement les prompts des branches (CM, Prospection, Rétention)
  • RAG juridique : upload des réglementations par pays, CGU des plateformes, RGPD → branche Compliance active
  • Rapport DG complet avec les 6 branches, insights, et décisions à arbitrer
Résultat : Le système s'auto-optimise. Le cerveau central orchestre tout. Tu pilotes en 5 min/jour.
PHASE 5 — SCALE Mois 3+
Ads + Influenceurs + Expansion internationale
  • Lancer les ads sur les meilleurs contenus organiques (Meta, TikTok, Google)
  • Recruter 5-10 micro-influenceurs VTC avec programme d'affiliation
  • Première expansion pays : dupliquer les workflows, changer langue + sources Apollo
  • Préparer les premiers datasets B2B licensing
  • Commande Telegram "/expansion belgique" → tout se configure automatiquement
Résultat : La machine tourne seule. Tu ne gères que la stratégie et les deals B2B majeurs.
